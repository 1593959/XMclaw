# Code of Conduct

XMclaw is a small, welcoming open-source project. This document sets the baseline for how we work together — keep it short, keep it human.

## Our pledge

We pledge to make participation in XMclaw a respectful and productive experience for everyone, regardless of background, experience level, or identity. We want this project to be a place where serious technical discussion happens without personal hostility.

## Expected behaviour

- Be respectful and constructive in issues, PRs, reviews, and chat.
- Critique code and ideas, not the people behind them. "This approach has a race condition" — fine. Personal attacks — not fine.
- Assume good intent. Most disagreements are misunderstandings, not malice.
- Accept feedback gracefully and offer it kindly.
- When you don't know, say "I don't know." When you're wrong, say "I was wrong." Both are signs of strength here.
- Keep discussion technical and on-topic. Off-topic threads get closed, not because the topic is bad, but because XMclaw is not the right venue.

## Unacceptable behaviour

- Personal attacks, insults, or sustained hostility toward another contributor.
- Harassment, intimidation, or threats — public or private.
- Publishing someone's private information without their consent.
- Sexualised content or advances in any project space.
- Sustained disruption of discussion (off-topic spam, repeatedly relitigating closed decisions, brigading).
- Other conduct that a reasonable maintainer would consider inappropriate in a professional open-source setting.

These apply to all project spaces: GitHub issues, PRs, Discussions, commit messages, release notes, and any synchronous channels we run.

## Scope

This Code of Conduct applies whenever you are representing XMclaw — contributing code, filing issues, commenting on PRs, posting in Discussions, or speaking on behalf of the project elsewhere.

## Enforcement

If you witness or experience a violation, report it via:

1. **GitHub private vulnerability disclosure** (works for any sensitive report, not just security): <https://github.com/1593959/XMclaw/security/advisories/new>
2. Or email a maintainer with `[XMclaw conduct]` in the subject (contact via the GitHub profile listed in `pyproject.toml`).

Reports are handled confidentially. We aim to acknowledge within 5 business days.

Maintainers may, at their discretion:

- Edit or remove offending content.
- Issue a private warning.
- Issue a temporary or permanent ban from project spaces.
- Revoke commit / triage rights.

We will explain the reasoning to the affected party. We will not name reporters without their consent.

## Maintainers' responsibilities

Maintainers are expected to enforce this Code fairly and consistently, including against themselves. A maintainer who violates the Code is held to the same standard as anyone else; a different maintainer handles the report.
